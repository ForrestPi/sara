Adaptive non maximal suppression algorithm (cf. [Multi-Image Matching using Multi-Scale Oriented Patches, Brown et al., CVPR 2005]).

This is the naive implementation which is quadratic in the number of features. Because of its complexity, it does not scale well.

Adaptive non maximal suppression is presented for the first time in: [Multi-Image Matching using Multi-Scale Oriented Patches, Brown et al., CVPR 2005]. It aims at discarding feature points so that the remaining features:

  • are as evenly spaced as possible;
  • are among the most distinctive set of points.

Let \((f_i)_{1 \leq i \leq N}\) be the set of feature points to filter. In the sequel, we denote:

  • by \(\mathbf{x}_i\) the position of feature \(f_i\),
  • by \(v_i\) the strength value of feature point \(f_i\).

For example, in the case of the Harris-Stephens corners, \(v_i\) are local maximum values of the Harris-Stephens cornerness function.

Specifically, the function computes for each feature \(f_i\) the suppression radius defined as:

\[ r_i = \min_{j \in I_i} \| \mathbf{x}_i - \mathbf{x}_j \|_2 \]

where \(I_i\) is the set of feature points \(f_j\) with values \(v_j\) stronger than value \(v_i\) of feature \(f_i\), i.e.,

\[ I_i = \{ j \in \{1,\dots, n\} \mid v_i < c_\textrm{robust} v_j \}. \]

Note that \(I_i\) can be empty, and in that then we set \(r_i = \infty\).

The intuition is the following. If \(f_i\) is close to stronger features \(f_j\) and has a value \(v_i\) much lower than their values \(v_j\), then \(f_i\) should be suppressed. It is indeed reflected by the fact that its suppression radius \(r_i\) is smaller than those of stronger features.

The adaptive non maximal suprression sorts feature points by suppression radius and so that we can keep those with the highest supression radius.
